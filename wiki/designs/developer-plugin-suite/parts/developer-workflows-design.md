# developer-workflows: the phase-gated loop

## Scope

Create `src/developer-workflows/` (`standalone: true`, `requires: []`; declares `capabilities: [setup, plan, work, review, release, bugfix]`) and extract the opinionated phase-gated process out of agentm:

- **Phase specs** — `harness/phases/{01-setup…05-release}.md` + `pipelines/bugfix.md` + `principles.md`.
- **Phase commands** — `/setup /plan /work /review /release /bugfix` (per-host adapters).
- **Sub-agents** — `explorer` + `evaluator` (the `evaluator` comes from the `#40` `developer` seed).
- **Hook** — `harness-context-session-start`.
- **`.harness/` scaffold templates** — PLAN / features / progress / init / verify, written at runtime by `/setup` (no reintroduced installer).
- **Workflow conventions** — `phases-one-per-session` · `work-single-task` · `work-loop-grouping` · `pre-approval-batch` · `subagents-read-only` · plan-md / status-report shapes · `wake-on-ci`.

The **`/review` phase is deliberately thin** — it orchestrates (gather diff, run deterministic gates) and dispatches a deeper adversarial pass only if available (wired in `auto-enable-runtime`). Built **parallel-run** against today's agentm (coexists with the baked-in copies; the agentm slim is V5 bucket ⑤, not here).

## Dependencies

- **enhances-schema** — to declare `capabilities:` (the targets `developer-safety` + `code-review` enhance) and to be a valid enhancee.

## Verification criteria

- Plugin loads on both hosts (`claude plugin list` / `agy plugin validate`).
- A phase command resolves and runs (e.g. `/plan` writes a `.harness/PLAN.md`); `/setup` writes the `.harness/` scaffold at runtime.
- The `harness-context-session-start` hook fires.
- `group.yaml` declares `capabilities:` including `review`.
- Generated byte-deterministically; `generate.py check` clean.
- **Works fully standalone** — no dependency on `developer-safety` or `code-review`.
- Dogfooded on agentm / crickets / sherwood.
