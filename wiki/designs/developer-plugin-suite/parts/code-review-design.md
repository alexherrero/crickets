# code-review: adversarial review (standalone + enhances `/review`)

## Scope

Create `src/code-review/` (`standalone: true`, `requires: []`, `enhances: [{group: developer-workflows, capability: review}]`) — adversarial code review usable on its own:

- **Sub-agents** — `adversarial-reviewer` + `adversarial-reviewer-cross` (cross-model via the Gemini CLI through `cross-review.sh`).
- **Hook** — `evidence-tracker` (grounds review in what was actually read).
- **Standalone command** — `/code-review <diff|PR>` — reviews any diff with no `/work` cycle.

Test-sanctity conventions (`tests-are-sacred`, `test-outcome-not-structure`, `verification-executable-first`) are **referenced, not owned** — they live in the future Testing bundle. The adversarial-reviewer contract holds: output is a failing test, a `file:line` defect, or an explicit NO-ISSUES finding (no prose-only critiques).

## Dependencies

- **enhances-schema** — provides `enhances:` + capability targeting.
- **developer-workflows** — the enhance target (its `review` capability).

## Verification criteria

- Plugin loads on both hosts.
- `/code-review <diff|PR>` runs **standalone** (no `/work` cycle) and produces the adversarial-reviewer contract output (failing test / `file:line` defect / NO ISSUES).
- `cross-review.sh` shells to the Gemini CLI for the cross-model pass; **graceful-skip** (or same-model fallback) when `gemini` is absent.
- `evidence-tracker` fires (PreToolUse).
- `enhances` target + `capability: review` validate in `lint_src.py`.
- Generated byte-deterministically; `generate.py check` clean.
- Dogfooded on agentm / crickets / sherwood.
